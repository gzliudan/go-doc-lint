# 常见问题（FAQ）

[English](FAQ.md) | 中文版

## 1. go-doc-lint 支持块注释（`/* */`）吗？

不支持。该工具仅检查函数声明上方使用 `//` 的 Go 文档注释，符合 Go 的标准注释风格。

## 2. 为什么我的注释正确，却被报告为不匹配？

工具会将注释的**首词**（区分大小写）与函数名比较。如果首词不同或以 `TODO:` 等标签开头，将判定为不匹配。

## 3. 如何仅扫描测试文件？

使用 `--test` 仅扫描 `*_test.go` 文件，使用 `--all` 扫描测试与非测试文件。

## 4. 为什么工具会跳过 vendor 和 .git 目录？

这些目录通常包含生成或外部代码，默认排除可减少噪音并提升性能。

## 5. 可以只扫描单个文件吗？

可以。直接传入文件路径：

```bash
./go-doc-lint.sh ./path/to/file.go
```

## 6. 输出为空但退出码是 0，是正常的吗？

是的。表示未发现不匹配。仅在错误或参数无效时返回非 0 退出码。

## 7. 如何处理生成代码？

生成文件通常包含 `Code generated by` 标记，建议仅扫描 `cmd/`、`pkg/`、`internal/` 目录。

## 8. 我的项目是 monorepo 或有多个 `go.mod`，该怎么扫？

建议分模块扫描：

```bash
./go-doc-lint.sh .
./go-doc-lint.sh ./services/user
./go-doc-lint.sh ./libs/common
```

## 9. go-doc-lint 支持 Windows 吗？

支持。使用 `go-doc-lint.ps1` 并确保 PowerShell 5.0+。
